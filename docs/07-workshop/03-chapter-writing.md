# Workshop Module 3: Chapter Writing / การเขียนบทวิทยานิพนธ์

> สอนเขียนบทวิทยานิพนธ์ด้วย /thesis-chapter และ thai_thesis_chapter

---

## ภาพรวม

ใน module นี้คุณจะได้เรียนรู้:
1. ใช้ /thesis-chapter command
2. เขียนบทด้วย master prompt
3. 4 โหมด: generate, revise, expand, condense
4. ลำดับการเขียนบท

---

## ทำความเข้าใจ /thesis-chapter

### Command

```
/thesis-chapter chapter={number} mode={mode}
```

### Parameters

| Parameter | จำเป็น | ค่าที่รองรับ | คำอธิบาย |
|-----------|--------|---------------|-----------|
| `chapter` | ✓ | 1-7 | หมายเลขบท |
| `mode` | ✓ | generate, revise, expand, condense | โหมดการทำงาน |
| `content` | - | ข้อความ | เนื้อหาที่มีอยู่ (สำหรับ revise/expand/condense) |
| `university` | - | chulalongkorn, kasetsart, kmutt, chiangmai, generic | มหาวิทยาลัย |

### 4 โหมด

| Mode | ใช้เมื่อ |
|------|----------|
| `generate` | เขียนบทใหม่ทั้งหมด |
| `revise` | แก้ไขเนื้อหาที่มีอยู่ |
| `expand` | ขยายเนื้อหาให้ละเอียดขึ้น |
| `condense` | ย่อเนื้อหาให้กระชับขึ้น |

## ขั้นตอนการเขียนบท

### Step 1: โหลด Skill

ก่อนเขียน ให้โหลด skill:

```
/skill thai-thesis-writing
```

### Step 2: เตรียม Master Prompt

สร้าง master prompt ที่มีข้อมูลครบถ้วน:

```markdown
# คุณคือนักเขียนวิทยานิพนธ์

คุณคือนักเขียนวิทยานิพนธ์ภาษาไทยระดับบัณฑิตศึกษา ผู้เชี่ยวชาญด้าน {สาขา}

ภารกิจ: เขียนวิทยานิพนธ์จากข้อมูลที่ให้

---

## ข้อมูลวิทยานิพนธ์

| รายการ | ข้อมูล |
|--------|--------|
| มหาวิทยาลัย | {มหาวิทยาลัย} |
| ชื่อนักศึกษา | {ชื่อ} |
| อาจารย์ที่ปรึกษา | {ชื่อ} |
| ชื่อวิทยานิพนธ์ | {ชื่อ} |

---

## กฎการเขียน

### ภาษา
- ภาษาไทยวิชาการ ตามหลัก ราชบัณฑิตยสถาน
- สรรพนาม: "ผู้วิจัย" (ไม่ใช้ "ผม", "ฉัน", "เรา")

### การอ้างอิง
- รูปแบบ: APA นาม-ปี
- ระบบปี: พ.ศ.
- ห้ามสมมติ — ใช้ [CITATION NEEDED]

### กฎ De-AI
- ค่า CV >= 30%
- ไม่มี banned patterns (นอกจากนี้, ก้าวสำคัญ, พลิกโฉม)

---

## โครงสร้างบทที่ {X}

{ใส่โครงสร้างบทที่ต้องการเขียน}

---

## ข้อมูลสำหรับเขียน

{ใส่ข้อมูลที่มี (technical specs, experiment data, etc.)}

---

## ข้อมูลที่ยังไม่มี

- [CITATION NEEDED]: {รายการที่ต้องหา}

---

## ข้อจำกัด

- ใช้เฉพาะข้อมูลที่ให้
- ห้ามเดาข้อมูล
- ความยาว: {X} หน้า
```

### Step 3: เรียก /thesis-chapter

```
/thesis-chapter chapter=1 mode=generate
```

แล้ววาง master prompt ตามด้วย

### Step 4: รอผลลัพธ์

ระบบจะสร้างเนื้อหาบทกลับมา

---

## ลำดับการเขียนบท (แนะนำ)

### ลำดับที่ 1: บท 3-6 (เนื้อหาหลัก)

```
3. ระเบียบวิธีวิจัย      ← วิธีการทำ
4. การวิเคราะห์และออกแบบ ← โครงสร้างระบบ
5. การดำเนินการ          ← การพัฒนา
6. ผลการวิจัย            ← ผลลัพธ์
```

**เหตุผล:** บทเหล่านี้มีข้อมูลชัดเจน (โค้ด, ข้อมูล) เขียนง่าย

### ลำดับที่ 2: บท 2 (ทฤษฎีและงานวิจัย)

```
2. ทฤษฎีและงานวิจัยที่เกี่ยวข้อง
```

**เหตุผล:** ต้องค้นอ้างอิงก่อน แต่เขียนได้เลย

### ลำดับที่ 3: บท 1, 7 (บทนำ/สรุป)

```
1. บทนำ
7. สรุปและข้อเสนอแนะ
```

**เหตุผล:** เขียนได้เมื่อมีเนื้อหาจากบทอื่นแล้ว

## สิ่งสำคัญ

### 1. ใช้ Master Prompt
- รวมทุกอย่างใน prompt เดียว
- ใส่ข้อมูลให้มากที่สุด
- ระบุ [TBD] และ [CITATION NEEDED] ชัดเจน

### 2. ทำ Waterfall ต่อบท
```
เขียน → De-AI → Review → Rewrite → ต่อบทถัดไป
```

### 3. ตรวจ De-AI หลังเขียน
- รัน De-AI scan ทุกครั้ง
- แก้ไขก่อนไปบทถัดไป
